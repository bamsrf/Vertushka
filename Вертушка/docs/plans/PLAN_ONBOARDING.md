# План онбординга — Вертушка

> Гибридный онбординг: экран приветствия + интерактивный тур по реальному UI, февраль 2026

---

## Общая идея

При первом входе в приложение пользователь видит:
1. **Экран приветствия** — полноэкранный слайд с градиентом, логотипом и кратким описанием
2. **Tooltip-тур** — 3 подсказки поверх реального интерфейса с подсветкой ключевых элементов

Онбординг показывается **один раз**, флаг сохраняется в хранилище.

---

## Фаза 1 — Стор и навигация

### 1.1 `useOnboardingStore` в Zustand
- **Файл**: `Mobile/lib/store.ts`
- **Что делаем**: Добавляем новый стор с полями:
  - `hasSeenWelcome: boolean` — показывать ли экран приветствия
  - `tourStep: number | null` — текущий шаг тура (null = тур не активен, 0–2 = шаги)
- **Методы**: `completeWelcome()`, `startTour()`, `nextStep()`, `skipTour()`, `completeTour()`, `checkOnboarding()`
- **Персистенция**: `expo-secure-store` (уже установлен), ключ `onboarding_complete`

### 1.2 Регистрация роута в навигации
- **Файл**: `Mobile/app/_layout.tsx`
- **Что делаем**: Добавляем `<Stack.Screen name="onboarding" />` в Stack

### 1.3 Редирект на онбординг
- **Файл**: `Mobile/app/(tabs)/_layout.tsx`
- **Что делаем**: После проверки авторизации — если `!hasSeenWelcome`, редирект на `/onboarding`

## Фаза 2 — Экран приветствия

### 2.1 Новый экран `app/onboarding.tsx`
- **Дизайн**:
  - Фон: `LinearGradient` (blue gradient из темы)
  - Центр: иконка винила + заголовок "Вертушка" + подзаголовок "Твоя виниловая коллекция"
  - 3 пункта с иконками:
    - `camera-outline` — "Сканируй штрихкод или обложку"
    - `search-outline` — "Ищи артистов, альбомы и релизы"
    - `library-outline` — "Управляй коллекцией и вишлистом"
  - Кнопка **"Начать"** (белая, скруглённая) — `completeWelcome()` + `startTour()` + `router.replace('/(tabs)')`
  - Ссылка **"Пропустить"** под кнопкой — пропускает и приветствие, и тур

## Фаза 3 — Tooltip-тур

### 3.1 Компонент `components/OnboardingOverlay.tsx`
- **Что делаем**: Overlay на весь экран поверх табов
- **Визуал**:
  - Полупрозрачный затемнённый фон (`rgba(0, 0, 0, 0.6)`)
  - "Вырезанное" окно (spotlight) вокруг целевого элемента
  - Glass-карточка с текстом подсказки (BlurView + `glassBg` из темы)
  - Кнопки **"Далее"** / **"Готово"** + ссылка **"Пропустить"**
- **Позиционирование**: Фиксированные координаты для каждого шага (layout детерминированный, вычисляется из стилей + safe area insets)

### 3.2 Шаги тура

| Шаг | Таб | Подсвечиваемый элемент | Текст подсказки |
|-----|-----|----------------------|----------------|
| 0 | Скан | Переключатель "Штрихкод / Обложка" | "Сканируй штрихкод или сфотографируй обложку — мы найдём пластинку автоматически" |
| 1 | Поиск | Строка поиска + кнопка фильтров | "Ищи по каталогу: артисты, альбомы, релизы. Используй фильтры по формату, стране и десятилетию" |
| 2 | Коллекция | Сегмент "В наличии / Вишлист" + папки | "Управляй коллекцией и вишлистом. Создавай папки, сортируй и следи за стоимостью пластинок" |

### 3.3 Переключение табов
- При нажатии "Далее" тур программно переключает активный таб через navigation API
- Последовательность: Скан → Поиск → Коллекция

### 3.4 Интеграция в табы
- **Файл**: `Mobile/app/(tabs)/_layout.tsx`
- **Что делаем**: Рендерим `<OnboardingOverlay />` поверх `<Tabs>`, если `tourStep !== null`

## Фаза 4 — Интеграция и флоу

### Полный путь пользователя

```
Запуск приложения
  → _layout.tsx: checkAuth() + checkOnboarding()
  → Не авторизован? → /(auth)/login
  → Авторизован + первый раз? → /onboarding
  → Авторизован + уже был? → /(tabs)

Экран /onboarding:
  → "Начать" → completeWelcome() + startTour() + router.replace('/(tabs)')
  → "Пропустить" → completeWelcome() + completeTour() + router.replace('/(tabs)')

Табы с тур-оверлеем:
  → Шаг 0: Скан — подсветка переключателя режимов
  → "Далее" → переключение на таб Поиск
  → Шаг 1: Поиск — подсветка строки поиска и фильтров
  → "Далее" → переключение на таб Коллекция
  → Шаг 2: Коллекция — подсветка сегментов и папок
  → "Готово" → completeTour(), оверлей исчезает
  → "Пропустить" (на любом шаге) → completeTour()
```

## Проверка

1. Очистить флаг онбординга в хранилище
2. Запустить приложение → должен появиться экран приветствия
3. Нажать "Начать" → табы + тултип на табе Скан
4. "Далее" → переключение на Поиск + тултип
5. "Далее" → переключение на Коллекцию + тултип
6. "Готово" → оверлей исчезает, обычное использование
7. Перезапуск → сразу в табы (онбординг не показывается)
8. "Пропустить" на любом шаге → всё закрывается корректно
